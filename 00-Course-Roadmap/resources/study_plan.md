# AI/ML Engineering Study Plan

This document outlines the recommended study paths for traversing the AI-ML-Engineering-Mastery repository. Depending on your time commitment, choose the track that fits your schedule.

## Track 1: The "Fast-Track" Bootcamp (12 Weeks)
*Intended for individuals treating this as a full-time bootcamp (30-40 hours/week).*

* **Weeks 1-2**: Module 01 to 05 (Python, Math, Probability, Statistics, Preprocessing)
* **Weeks 3-4**: Module 06 to 08 (Linear Algebra, Calculus, Regression Algorithms)
* **Weeks 5-6**: Module 09 to 11 (Classification, Clustering, Dimensionality Reduction)
* **Weeks 7-8**: Module 12 to 14 (Neural Nets from Scratch, PyTorch/TF Foundations, Evaluation)
* **Weeks 9-10**: Module 15 to 18 (Feature Eng, Ensemble, Multi-Model, RAG)
* **Weeks 11-12**: Module 19 to 24 (Dashboards, MLOps, Capstones, Interview Prep)

## Track 2: The "University Semester" Pacing (6 Months)
*Intended for part-time learners who want to deeply absorb the math (10-15 hours/week).*

* **Month 1: The Foundations**
  * Module 01: Python For AI
  * Module 02: Mathematics For ML
  * Module 03: Probability
  * Module 04: Statistics
* **Month 2: Data & Algebra**
  * Module 05: Data Preprocessing
  * Module 06: Linear Algebra
  * Module 07: Calculus For ML
* **Month 3: Traditional Machine Learning**
  * Module 08: Regression Algorithms
  * Module 09: Classification Algorithms
  * Module 10: Clustering
  * Module 11: Dimensionality Reduction
* **Month 4: Deep Learning Foundations**
  * Module 12: Neural Networks From Scratch
  * Module 13: Deep Learning Foundations
  * Module 14: Model Evaluation
* **Month 5: Advanced & Systems**
  * Module 15 & 16: Feature Eng & Ensembles
  * Module 17 & 18: Multi-Model & RAG
* **Month 6: Production & Projects**
  * Module 19 to 21: UI & MLOps
  * Module 22 to 24: Capstones & Career

## Tips for Success
1. **Always calculate manually first.** If a module has a mathematical derivation (like computing the gradient for backpropagation), do it on paper before running the Python script.
2. **Read the library source.** We discuss how Scikit-Learn works under the hood. Don't be afraid to read their actual GitHub repository.
3. **Finish the assignments.** The `assignments/` folder contains problems without code. Try to solve them before looking at the `solutions/` folder.
